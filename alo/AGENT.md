# 目标

由于'xuance'这个RL库内置的算法和环境都是模块化的，对于想要专注算法学习的初学者很不友好。通常，单个文件更容易理解。
所以你需要根据一份输入文件test_xuance.py，完整提取这份文件运行后的所有代码到一个python文件中。

# 步骤

## 步骤1

运行`conda activate xuance`激活环境

## 步骤2

当前目录下创建OUT.py文件

## 步骤3

读取当前目录下的test_xuance.py文件

## 步骤4

从test_xuance.py开始，递归分析代码实际运行后import的所有模块或者文件清单，你不需要急于读取它们的内容

## 步骤5

**如果**：这些import的模块或者文件是从'xuance'这个库中导入的，则导航到其所在目录，并读取文件内容前30行，然后跳转步骤4。你需要始终保留总的模块或者文件清单的顺序。

**否则**：从当前文件内容中提取实际用到的类或者函数，将其需要依赖的import语句追加写入OUT.py文件中，注意处理相对路径的import情况，例如'from .run_drl import RunnerDRL'。然后将这些类或者函数追加写入OUT.py文件中，注意去掉注释。

## 步骤6

当你处理完了步骤4和步骤5相关的任务，你需要按照以下步骤检查、整理OUT.py文件。

首先，确保test_xuance.py文件的内容在OUT.py文件末尾

然后，确保所有import 语句都OUT.py文件的顶部

接着，尝试运行OUT.py文件（或者按序分析代码，不运行），检查是否有错误。如果有，尝试最小幅度修复。因为这通常是你在拷贝代码时没有处理文件的相对依赖关系或者路径问题导致的。如果涉及到从yaml或者其他类型的文件中读取数据，则你需要拷贝这些文件到当前目录下。

然后，去除所有没有用到的代码块、类、函数。

最后，根据代码的功能，优化代码，去除冗余逻辑。

评价你的结果，并进行必要的调整。

# 注意事项

1. 确保**只读取**当前目录下的test_xuance.py文件以及其依赖的库xuance下的源码文件。xuance这个库的位置在'D:\.conda\envs\xuance_env\Lib\site-packages\xuance'
2. 确保**只修改**你创建的文件。
3. 不要尝试**一次性读取所有文件的内容**，你可以按照顺序依次读取。
4. 涉及到的非.py文件，例如yaml、json等，你只需要拷贝到当前目录下就可以了，无需读取。
5. 对于**不属于xuance这个库**的import的模块或者文件，你只需要照抄import语句即可，不需要跳转读取。
